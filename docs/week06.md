## Definition of Topics  

* **Wireframes** are basic visual layouts that outline the structure of a page or interface without detailed design elements.  

* **User Flows** represent the path a user takes to complete a specific task within a product or system.  

* **Artifacts** are visual outputs such as sketches, diagrams, or mockups used to communicate design ideas.  

* **User Flow Diagrams** are visual representations that map out each step a user takes when interacting with a system.  

* **Low Fidelity Mockups** are simple, rough versions of a design that focus on layout and functionality rather than visual details.  

## Wireframing Process

Wireframing is used to establish the basic structure of a design before adding visual detail. It typically involves:  

1. Defining layout structure – arranging key elements such as headers, content areas, and navigation.  

2. Focusing on functionality – emphasizing how users will interact with the interface rather than how it looks.  

3. Iterating quickly – making rapid changes to explore different layout options and improve usability.  

## User Flow Development

User flows help designers understand and improve how users complete tasks. This process typically includes:  

1. Identifying user goals – determining what the user wants to accomplish.  

2. Mapping steps – outlining each action the user takes to reach their goal.  

3. Analyzing efficiency – reviewing the flow to remove unnecessary steps and improve the experience.
